# Parallax Roadmap

Date: 2026-06-09

This roadmap overrides old notes, handoffs, history files, and idea-bank
documents. Those files are archived as historical context, not active build
guidance.

## Current Product Spine

1. Household / Plan: input the client household and assumptions.
2. Scenarios: compare planning choices on the same market draws.
3. Sequencing: run the same plan through real markets.
4. Cash-flow detail: prove the result year by year.

## Current Engine Status

Trust these only after verification against the live code and tests:

- Real-return block-bootstrap Monte Carlo.
- Same-path scenario comparison.
- Taxable / traditional / Roth sleeves.
- Basis tracking.
- Withdrawal sequencing.
- Required minimum distributions.
- Savings splits across sleeves.
- Timed goals, expenses, income, and liabilities.
- Property mortgage handling.
- Override-based asset sales.
- Healthcare separated from lifestyle spend.
- Historical sequencing.

Known modeling gaps:

- Full progressive federal tax engine.
- Social Security survivor benefits.
- LTC escalation above CPI.
- Goal-priority funding logic.
- One-time inflow UX and scenario framing.
- Full real-assets UI.
- Solve-For solo-vs-combined display clarity.

## Build Next

Low-risk, high-value views of existing engine output:

1. Failure Anatomy: click a success ring and show when and why failures happen.
2. Scenario Receipt: clear before/after delta summary for a scenario.
3. Sequence Pressure Strip: first 10 retirement years with return, withdrawal,
   and ending balance.
4. Funding Bridge: show years carried by portfolio before Social Security or
   pension begins.
5. Rename Net Worth to Plan / Household.
6. Duplicate Scenario.

## Build Later

Work that needs larger engine or UX scope:

1. Full tax engine.
2. Multi-variable explainable constraint solver.
3. Decision Surface heatmap.
4. Pension claim-age sweep.
5. Real assets input and sale workflow.
6. Strategy Fork: compare strategies against one fixed bad market.

## Skip

Do not rebuild these unless a new product decision changes the scope:

- Fancy cash-flow river / Sankey.
- Resilience Matrix as a new tab.
- Reverse-the-timeline view.
- Allocation as a solve lever without careful framing.
- More dashboard tiles that do not reveal new truth.
- Any UI math not generated by the engine.

## Open Product Decisions

- Should liabilities be base-plan inputs, per-scenario choices, or both?
- Should Social Security be framed as lifetime dollars / insurance rather than
  success-rate movement?
- Should goal priority remain conversation-only or eventually drive funding
  order?
- How should solver results display solo answers without implying all answers
  are applied together?
